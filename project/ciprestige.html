Design Notes

Every page currently has a Save, Back, and Next button
- Implement a “clear” button for the IP questionnaire. Users would expect a reset button to clear the answers on the form
- Can’t have two reset buttons that do different things - would cause confusion on the part of the user
Mosaic: ESG or Non-ESG portfolio suggestions

INVESTMENT OPTIONS
After selecting “Guided Portfolio" or “Individual Investments", user has no way of knowing that the next step is INVESTMENT SELECTION. There is a Next’ button that can’t be used until the user has gone through the investment selection process.

Suggestions
- rename Next button to Investment Selection
- highlight INVESTMENT SELECTION when next button is clicked

MOSAIC/PS/PSS SELECTION
- Images or no images?
- make selection obvious including on mobile
- when a selection is made, reduce the opacity of non-selected options
- include highlighted border on the selected option

QUESTIONNAIRE
- Currently, the Back button would theoretically take the user one page back - the ps/pss mosaic selection page. No need for the Reset button to do the same task since that would be redundant.
- Rename the Reset button, should have the same current functionality - clearing radio checkboxes Regarding ESG or Non-ESG for the Mosaic selection:
- when SKIP QUESTIONNAIRE is selected, highlight a “ESG or Non-ESG" choice

SELECTED/SUGGESTED PORTFOLIO
- The only page with a Reset button
- Reset button clears the choices and takes the user back to the ps/pss/mosaic selection
- Hovering on the Reset button should tell the user what the button does

Process

Due to the exhaustive nature of the project, I decided to have the dev environment on my local. This meant that I could implement and experience the changes I wanted in real-time.

One of the major requests from stakeholders was a new selection screen to accomodate the new experience they were hoping to implement. Wireframes anf userflows are a very important part of the user experience design process. I went through a few itirations of this particular screen before landing on something that I felt worked well on desktop and could easily translate to mobile screens.
